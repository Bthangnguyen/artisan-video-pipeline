# 🚀 Artisan Video Pipeline (V3)

The **Artisan Video Pipeline** is a modern, 3-stage high-fidelity AI video generation pipeline designed to cleanly separate logic generation, human-guided editing, and video rendering.

## 🌟 System Architecture

At its core, this monorepo consists of 4 tightly coupled modules governed by a strict Zod schema definition:

```text
[AI Backend / Agent] ---> (Generates) ---> draft_state.json
                                             |
                                             v
[The Cockpit (Next.js)] <--- (Loads) --- draft_state.json
       |
       +---> (Human Tweak: timing, assets, camera springs)
       |
       +---> (Saves) ---> final_state.json
                             |
                             v
[Remotion Server] <--- (Reads) --- final_state.json ---> (Renders) ---> out.mp4
```

## 📦 Monorepo Structure

Built on **TurboRepo** and **npm workspaces**, the system is split into independent but interoperable packages:

- `packages/types`: Global data contracts and Zod schemas (`ProjectState`, `SceneNode`, `CameraPath`). The source of truth across all apps.
- `packages/assets`: Centralizes raw assets, specifically static SVGs and corresponding enumeration mappings.
- `apps/web-gui` ("The Cockpit"): A Next.js dashboard featuring state management via `zustand`. It lets human reviewers inspect generated JSON states, scrub timelines, tweak offsets, adjust camera spring physics, and export the confirmed state.
- `packages/remotion-engine`: The core rendering mechanism. It hosts Remotion React components (including `CinematicCamera`) to generate the deterministic MP4 buffer, coupled with an Express.js Render API server running FFmpeg sub-processes.

## 🚀 Getting Started

### 1. Install Dependencies
Switch to the monorepo root and install dependencies. Because we rely on `npm workspaces`, everything links automatically!
```bash
npm install
```

### 2. Start "The Cockpit" (Human-in-the-Loop Web GUI)
```bash
cd apps/web-gui
npm run dev
```
Navigate to `http://localhost:3000`. You can upload the default `dummy/final_state.json` (located in the Remotion package) to preview the interactive workflow.

### 3. Start the Render API (Remotion Server)
```bash
cd packages/remotion-engine
npm run server
```
This boots up an Express instance on port `3001` that awaits POST payloads containing a valid `ProjectState`.

### 4. Preview Rendering in Studio
If you want to debug the video directly using the Remotion timeline editor:
```bash
cd packages/remotion-engine
npm start
```

## 🛠 Features

* **Cinematic Spring Physics:** Remotion automatically interpolates camera waypoints seamlessly using spring physics (mass, damping, stiffness).
* **Deterministic Timing:** Nodes exist inside Remotion `<Sequence>`s strictly defined by frames (e.g. `startFrame + offsetFrames`), providing absolute frame-perfect mapping against AI text-to-speech audio.
* **Lock Mechanisms:** The `/api/render` API implements an `isRendering` thread block preventing server overload from multiple render invocations.
